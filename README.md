# 群违规检测插件

一个功能强大的 AstrBot 插件，用于自动检测群聊中的违规内容，包括违规图片、刷屏行为、骂人内容、广告、链接和群号推广等。

## 功能特性

### 六大检测功能

#### 1. 图片违规检测
- 自动检测群聊中的图片
- 使用 AI 视觉模型分析图片内容
- 检测色情内容（裸露、性行为、性暗示等）
- 检测擦边内容（暴露服装、性感姿势等）
- 自动撤回违规图片
- 自动禁言违规用户

#### 2. 刷屏检测
- 检测用户在短时间内发送大量消息
- 可自定义刷屏阈值和时间窗口
- 自动撤回刷屏消息
- 自动禁言刷屏用户

#### 3. 骂人检测
- AI检测模式：使用 AI 智能识别骂人内容
- 关键词检测模式：使用预设关键词匹配
- 支持动态添加/删除关键词
- 自动撤回骂人消息
- 自动禁言骂人用户

#### 4. 广告检测
- 自动检测广告关键词（加群、加微信、代练、外挂等）
- 预设24个常见广告关键词
- 自动撤回广告消息
- 自动禁言广告用户

#### 5. 链接检测
- 自动检测消息中的链接
- 支持多种链接格式（http/https/www等）
- 自动撤回链接消息
- 自动禁言发送链接的用户
- 默认关闭（需手动开启）

#### 6. 群号推广检测
- 自动检测群号推广行为
- 检测关键词：进群、加群、群号、入群、拉群、建群
- 智能识别群号（5-12位数字）
- 支持多种格式（进群；12345678、加群：12345678等）
- 自动撤回推广消息
- 自动禁言推广用户

### 白名单功能
- 白名单用户不受任何检测限制
- 支持动态添加/删除白名单用户
- 查看白名单列表
- 白名单用户优先级高于管理员

### 违规统计功能
- 统计每个用户的违规次数
- 分类统计（图片/刷屏/骂人/广告/链接）
- 查看单个用户违规详情
- 查看整体违规概览

###昵称检测
— 可识别昵称违规词
— 可禁用昵称检测


## 安装方法

1. 将插件文件夹放入 AstrBot 的 `data/plugins/` 目录
2. 重启 AstrBot
3. 在管理面板配置 API 参数

## 配置说明

### 必需配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| API站点地址 | AI API 的 chat/completions 端点 | `https://api.siliconflow.cn/v1/chat/completions` |
| API Key | API 密钥 | `sk-xxxxxxxx` |
| 模型名称 | 支持视觉的模型 | `Qwen/Qwen2-VL-72B-Instruct` |

### 可选配置

#### 禁言时长配置（统一默认 600 秒）
- 图片违规禁言时长：默认 600 秒（10分钟）
- 刷屏禁言时长：默认 600 秒（10分钟）
- 骂人禁言时长：默认 600 秒（10分钟）
- 广告禁言时长：默认 600 秒（10分钟）
- 链接禁言时长：默认 600 秒（10分钟）
- 群号推广禁言时长：默认 600 秒（10分钟）

#### 图片检测配置
- 检测阈值：违规判定阈值（0-1），默认 0.7
- 检测色情：是否检测色情内容，默认开启
- 检测擦边：是否检测擦边内容，默认开启

#### 刷屏检测配置
- 启用刷屏检测：默认开启
- 刷屏阈值：默认 5 条消息
- 刷屏检测时间窗口：默认 10 秒

#### 骂人检测配置
- 启用骂人检测：默认开启
- 使用AI检测骂人：默认开启
- 骂人关键词列表：预设关键词列表

#### 广告检测配置
- 启用广告检测：默认开启
- 广告关键词列表：预设24个关键词

#### 链接检测配置
- 启用链接检测：默认关闭
- 链接禁言时长：默认 600 秒

#### 群号推广检测配置
- 启用群号推广检测：默认开启
- 群号推广禁言时长：默认 600 秒

#### 通用配置
- 监控群组列表：留空表示监控所有群组
- 白名单用户列表：不受检测限制的用户
- 管理员豁免：管理员发送的内容跳过检测，默认开启
- 违规通知：检测到违规时发送通知，默认开启

## 工作原理

### 图片检测流程
1. 监听群消息，检测是否包含图片
2. 下载图片并转换为 base64
3. 发送到 AI API 进行分析
4. 解析 AI 返回的 JSON 结果
5. 根据阈值判断是否违规
6. 执行撤回和禁言操作

### 刷屏检测流程
1. 记录每个用户的消息时间戳
2. 统计时间窗口内的消息数量
3. 超过阈值则判定为刷屏
4. 执行撤回和禁言操作

### 骂人检测流程

#### AI检测模式
1. 提取消息文本内容
2. 发送到 AI API 进行分析
3. 解析 AI 返回的判断结果
4. 执行撤回和禁言操作

#### 关键词检测模式
1. 提取消息文本内容
2. 遍历关键词列表进行匹配
3. 匹配成功则判定为骂人
4. 执行撤回和禁言操作

### 广告检测流程
1. 提取消息文本内容
2. 遍历广告关键词列表进行匹配
3. 匹配成功则判定为广告
4. 执行撤回和禁言操作

### 链接检测流程
1. 提取消息文本内容
2. 使用正则表达式匹配链接
3. 匹配成功则判定为链接
4. 执行撤回和禁言操作

### 群号推广检测流程
1. 提取消息文本内容
2. 检测是否包含推广关键词（进群、加群等）
3. 使用正则表达式匹配群号（5-12位数字）
4. 两个条件都满足则判定为群号推广
5. 执行撤回和禁言操作

## 注意事项

### 权限要求
- 机器人必须有群管理员权限才能撤回消息和禁言用户
- 如果机器人是群主，确保在 NapCat 中正确配置权限

### API 配置
- 推荐使用支持视觉的模型，如：
  - Qwen/Qwen2-VL-72B-Instruct（推荐）
  - deepseek-ai/deepseek-vl2
  - gpt-4o
- 确保 API 站点地址正确，通常是 `/v1/chat/completions` 端点

### 性能优化
- 图片检测会消耗较多 API 调用
- 建议合理设置检测阈值，避免误判
- 可以通过 `enabled_groups` 配置只监控特定群组
- 使用白名单功能豁免信任用户

## 常见问题

### Q: 插件没有识别到图片？
A: 检查日志中是否有 `[群违规检测] 检测到 X 张图片` 的输出。如果没有，可能是图片格式不支持。

### Q: 消息没有被撤回？
A: 
1. 确认机器人有群管理员权限
2. 检查日志中是否有撤回成功的提示
3. 某些客户端可能有防撤回功能，实际消息已被撤回

### Q: AI 检测不准确？
A: 
1. 调整检测阈值（降低阈值会更严格）
2. 尝试不同的模型
3. 可以自定义检测提示词

### Q: 刷屏检测误判？
A: 调整刷屏阈值和时间窗口，例如设置为 10 条/20 秒会更宽松。

### Q: 如何关闭某个检测功能？
A: 在管理面板中找到对应的开关配置项，设置为 false 即可。

### Q: 白名单用户为什么还会被检测？
A: 检查白名单配置是否正确，确保 QQ 号格式正确（纯数字）。
